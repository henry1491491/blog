Python Django 環境設定

此筆記僅為了紀錄架設 Python 網站過程

建立虛擬環境（virtualenv）

為了讓不同 Python 專案之間環境乾淨，或是避免影響到全域環境，通常很多人會建立虛擬環境來進行開發

pip3 安裝虛擬環境（因為使用 Python3, macOS）

打開 zsh 配置檔（因為我的 shell 使用的是 zsh）

加上以下幾行在結尾

  • VIRTUALENVWRAPPER_PYTHON 對應到 python3 的位置
  • /usr/local/bin/virtualenvwrapper.sh 這行也要確定是否該路徑下存放 virtualenvwrapper.sh 是對的
  • 以上兩者確認路徑方式可透過

設定完成後，重置設定

現在只要下指令，便能創建一個虛擬環境

然後我們可以在終端機看到提示開頭的 () 裡面就是剛剛創建的虛擬環境

常用的命令：

  • deactivate — 退出當前的 Python 虛擬環境
  • workon — 列出可用的虛擬環境
  • workon name_of_environment — 啟動指定的Python 虛擬環境
  • rmvirtualenv name_of_environment — 刪除指定的環境
